The main Basalt Module itself, which contains everything inside the project.

All Types

basalt.player.AudioLoadHandler

Class which loads sources based on provided identifiers asynchronously.
basalt.server.AudioSender

The AudioSendHandler used by Magma to actually send audio in the first place.
basalt.util.AudioTrackUtil

The utility class which converts AudioTracks to easily-transportable data and vice versa.
basalt.player.BasaltPlayer

A player class which contains methods that react to lavaplayer events. Represents a unique audio stream for a specific guild.
basalt.server.BasaltServer

The main entry class for Basalt which is responsible for every single last part of the application. As of v4, this class handles most things, such as HTTP and WebSocket connections and timers, itself without shipping those tasks off to other classes.
basalt.exceptions.ConfigurationException

Represents an issue with the Basalt configuration file, most commonly thrown when a certain field is null.
basalt.server.ErrorResponse

Inline class meant to restrict types without performance hits. This is all optimized heavily by Kotlin's compiler.
basalt.server.ErrorResponses

An object containing responses that are returned in Dispatched ERROR Responses, as they're written here.
basalt.player.LoadResult

A result class which represents a singular, successful load result.
basalt.server.MessageType

An inline class which represents the type of message response sent back to clients. Inline classes are optimized heavily by the Kotlin compiler, so no performance hits occur.
basalt.server.MessageTypes

An object containing a bunch of message types that indicate the type of event being sent back to clients.
basalt.exceptions.MissingContextException

Thrown when an "impossible" event occurs where a connected socket doesn't have an associated basalt.player.SocketContext.
basalt.player.ResultStatus

Enum which corresponds to how a source loaded.
basalt.player.SocketContext

A SocketContext meant to associate a User ID with a WebSocket connection and a map of players as well as the ID of a timer to be scheduled for resume purposes when the connection closes.
